На странице поиска партнеров, даже при выбранных фильтрах, всегда выводится сообщение "Ничего не найдено". Поиск не работает ни по одной категории ("IT и Технологии", "Строительство" и пр.), результаты не отображаются.​

При просмотре профиля компаний часть полей (Телефон, Email, Количество сотрудников, Среднегодовая выручка, Год основания) всегда пустая, даже для верифицированных компаний, что препятствует полноценной проверке данных.​

Нет возможности отправить массовый запрос: после выбора компаний и заполнения формы нет кнопки для завершения отправки либо кнопка неактивна.​

В разделе сообщений нельзя начать диалог — отсутстует возможность выбрать и отправить первое сообщение, только написано "Выберите компанию для начала диалога" (UI неинтерактивен).​

На страницах профиля количество активностей, просмотров, отправленных запросов и рейтинг всегда показывают 0, даже при попытках совершить действия.​

При восстановлении пароля после ввода email кнопка "отправить инструкции" может быть неактивна, нет уведомления об успехе/ошибке, кроме отдельной страницы.​

В разделе "Профиль компании" все поля отображаются, но сохранение изменений не демонстрирует явного результата, нет всплывающего уведомления об успехе.​

В разделе настроек уведомления невозможно включить — переключатели неактивны.


Детальный список необходимых доработок на основе функционального и UI тестирования сайта:

1. Поиск партнеров и отображение компаний
Исправить логику поиска: фильтры должны возвращать релевантные результаты, а не "Ничего не найдено".​

Реализовать отображение компаний для всех категорий; обеспечить корректную работу сортировки и фильтрации.

Добавить блоки для отображения подробных данных (контактные данные, активность, ссылки).

2. Формы и валидация

Проверять обязательные поля (ИНН, ОГРН, телефон) в форме регистрации и при редактировании профиля компании.

3. Массовые запросы
Сделать кнопку отправки активной только при заполнении всех необходимых полей.

После отправки массового запроса показывать отчет/уведомление об успехе либо ошибке отправки.

Реализовать логику выбора получателей — отображать выбранные компании, предоставлять обратную связь.

4. Сообщения
Активировать UI для начала нового диалога: реализовать список доступных компаний для выбора, кнопку "Начать переписку".

После отправки сообщения отобразить его статус и дать обратную связь (отправлено/ошибка).

5. Личный кабинет и профиль компании
Доработать отображение активностей, рейтинга, просмотров: значения должны изменяться при новых действиях.

Реализовать всплывающее уведомление/индикаторы успеха при сохранении изменений профиля.

Заполнить/отобразить все поля (Год основания, телефон, email, количество сотрудников, оборот) для полной карточки компании.​

6. Восстановление пароля
Исправить работу кнопки "отправить инструкции" при восстановлении пароля — сделать ее всегда активной, показывать уведомление о результате.​

После успешного восстановления пароля — направлять пользователя на страницу входа с соответствующим предупреждением.

7. Управление уведомлениями
Сделать настройки push- и email-уведомлений рабочими: переключатели должны изменять состояния, сохранять выбор пользователя.

Показывать статус включения уведомлений.

8. Общие UI/UX доработки
Добавить обратную связь для всех пользовательских действий: саппорты, тосты об успехе/ошибке, загрузки.

Проверить корректность навигации по всем разделам, устранить неработающие ссылки.

Оптимизировать отображение форм и их заполнение на мобильных устройствах.

Добавить подробные описания ошибок при некорректном вводе.
